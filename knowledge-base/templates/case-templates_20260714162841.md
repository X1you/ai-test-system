# 用例编写模板

## 正向测试模板

### 标准操作流程
```
前置条件：用户已登录，具备操作权限
测试步骤：
1. 准备测试环境，确认前置条件满足
2. 执行目标操作（填写数据/点击按钮/发送请求）
3. 确认操作成功
4. 检查返回结果是否正确
预期结果：操作成功，结果符合预期
```

### 查询类操作
```
前置条件：用户已登录，系统中有可查询的数据
测试步骤：
1. 进入查询页面/模块
2. 输入查询条件
3. 执行查询操作
4. 检查查询结果是否正确
预期结果：返回符合条件的正确数据
```

### 创建类操作
```
前置条件：用户已登录，具备创建权限
测试步骤：
1. 填写/选择必要的信息
2. 点击提交/保存按钮
3. 确认操作成功提示
4. 检查返回结果是否正确
预期结果：创建成功，数据正确返回
```

## 负向测试模板

### 空数据
```
前置条件：系统处于可测试状态
测试步骤：
1. 保持必填字段为空
2. 提交/确认操作
3. 检查系统提示
预期结果：系统提示必填项不能为空
```

### 非法数据
```
前置条件：系统处于可测试状态
测试步骤：
1. 在输入字段中输入非法数据：XXX
2. 提交操作
3. 检查系统响应
预期结果：系统拒绝并给出正确的错误提示
```

## 边界测试模板

### 数值边界
```
前置条件：测试环境已就绪
测试步骤：
1. 确认目标字段的边界值（最小值/最大值）
2. 设置测试数据为边界值
3. 执行操作
4. 检查系统是否正确处理
预期结果：边界值正确处理，不报错
```

## 异常测试模板

### 超时异常
```
前置条件：正常的测试环境
测试步骤：
1. 模拟网络超时/服务无响应
2. 执行操作
3. 观察系统处理超时的方式
预期结果：系统给出超时提示，有重试机制
```

## 安全测试模板

### 越权访问
```
前置条件：存在不同权限级别的用户
测试步骤：
1. 使用低权限用户 A 登录
2. 尝试访问高权限用户 B 的数据或功能
3. 验证系统是否阻止
预期结果：系统阻止越权访问并给出提示
```

### 注入测试
```
前置条件：系统有输入框/参数接口
测试步骤：
1. 在输入框/参数中注入恶意载荷
2. 提交请求
3. 检查是否被过滤
预期结果：注入被拦截，不泄露错误信息
```
