# 线上坑点记录

## 测试用例生成系统开发踩坑

### 1. check_has_results 中文编码 SyntaxError
**模块：** pipeline.py
**严重度：** 高
**发现时间：** 2026-07-14
**现象：** f-string 中包含中文（"执行结果"）传给 `python -c` 参教时，macOS 默认编码导致 SyntaxError: Non-UTF-8 code
**原因：** subprocess 调用 `-c` 参数时中文字符未正确编码
**修复：** 
1. 优先直接 import openpyxl（不用 subprocess）
2. 回退方案用 Unicode 转义（\\u6267\\u884c\\u7ed3\\u679c）代替中文字符

### 2. 执行结果列匹配错误
**模块：** pipeline.py
**严重度：** 高
**现象：** 模糊匹配 `"结果" in header` 误匹配到"预期结果"列而非"执行结果"列
**原因：** 匹配逻辑先匹配短字符串，导致"预期结果"优先于"执行结果"
**修复：** 精确匹配优先（`header == "执行结果"`），模糊匹配兜底

### 3. Step 4 覆盖已有执行结果的 Excel
**模块：** pipeline.py
**严重度：** 中
**现象：** 续跑 pipeline 时 Step 4 重新生成 Excel，覆盖了已填写执行结果的版本
**原因：** Step 4 没有检查 Excel 是否已有执行结果
**修复：** 生成前调用 check_has_results，有结果时跳过

### 4. 测试点生成正向占比过高
**模块：** test-points SKILL.md
**严重度：** 中
**现象：** 生成的测试点中正向占比 79.5%，负向/边界/异常严重不足
**原因：** AI 倾向于生成正向测试点，缺少维度平衡约束
**修复：** SKILL.md 中添加维度平衡约束（正向 ≤ 30%）

### 5. 步骤描述笼统
**模块：** generate-excel.py
**严重度：** 中
**现象：** 生成的用例步骤描述为"按照业务流程执行: [标题]"，无法直接执行
**原因：** 步骤模板过于通用，缺少基于操作语的细化
**修复：** 重写步骤模板，按操作类型（创建/查询/删除/编辑等）生成具体步骤

### 6. P0 优先级缺失
**模块：** generate-excel.py
**严重度：** 中
**现象：** 所有 44 个用例中没有一个 P0 优先级
**原因：** 优先级分配规则只识别了"下单/付款/退款/登录"等电商关键词
**修复：** 扩展核心操作关键词列表，增加"创建/校验/主流程/关键"等

### 7. 优先级分配不合理
**模块：** generate_xmind.py
**严重度：** 低
**现象：** XMind 和 Excel 使用不同的优先级分配逻辑
**原因：** 两个脚本独立开发，分配规则未对齐
**修复：** 统一两套脚本的分配逻辑和关键词表
