# 测试报告生成 Skill

根据测试用例执行结果，自动生成结构化测试质量报告，包含总体概览、模块通过率分布、失败用例分析和风险评估。

## 触发条件

当用户说以下内容时，使用本 Skill：
- "生成测试报告"
- "测试执行报告"
- "测试质量报告"
- "分析测试结果"
- 带有 "report"、"报告"、"测试结果" 等关键词

## 执行步骤

### 1. 读取测试执行结果文件

使用 `read_file` 工具读取测试用例文件（已完成执行的 Excel 或 Markdown），支持：
- Excel 文件（`.xlsx`）— **必须包含「执行结果」列**
- Markdown 文件（`.md`）— 必须包含执行结果信息

**执行结果的合法取值（中英文均可）：**
| 中文 | 英文 | 缩写 | 含义 |
|------|------|------|------|
| 通过 | Pass / Passed | ✅ | 测试通过 |
| 失败 | Fail / Failed | ❌ | 测试失败 |
| 阻塞 | Block / Blocked | ⛔ | 被阻塞无法执行 |
| 跳过 | Skip / Skipped | ⏭️ | 主动跳过 |
| 未执行 | N/A / Not Run | — | 尚未执行 |

> ⚠️ 如果执行结果列为空或全部为「未执行」，提示用户先执行测试。

### 2. 运行报告生成脚本

使用 Hermes venv Python 运行 `scripts/generate_report.py`：

```bash
HERMES_PYTHON="$HOME/.hermes/hermes-agent/venv/bin/python"
$HERMES_PYTHON scripts/generate_report.py <testcases.xlsx> -o test_report.md
```

**参数说明：**
- `<testcases.xlsx>` — 已执行完成的测试用例文件（必需）
- `-o / --output` — 输出报告路径（可选，默认 `test_report.md`）
- `-r / --requirements` — 需求分析文件（可选，用于计算需求覆盖率）

### 3. 向用户汇报

**汇报格式：**
```markdown
✅ 测试报告生成完成！

📊 总体概览：
- 总用例数：N 个
- 通过：N 个（XX.X%）
- 失败：N 个
- 阻塞：N 个
- 跳过：N 个

📈 模块通过率：
- 模块A：XX.X%
- 模块B：XX.X%
- ...

🔍 失败分析：N 个失败用例
⚠️ 风险评估：高风险 N 个，中风险 N 个

📁 生成的文件：
- test_report.md - 测试报告（N KB）

💡 下一步：
1. 检查报告中的失败用例分析
2. 根据风险评估优先修复高风险问题
3. 将报告同步给团队
```

## 输入参数

在执行时需要确认：
- 测试用例文件路径（必需）— 已执行完成的 Excel/Markdown 文件
- 输出目录（可选，默认为当前目录）
- 需求分析文件（可选，用于需求覆盖率计算）
- 输出文件名（可选，默认为 `test_report.md`）

## 报告内容结构

生成的 `test_report.md` 包含以下部分：

### 1. 总体概览
- 总用例数、通过/失败/阻塞/跳过数量
- 通过率、执行率
- 质量评级（优秀/良好/中等/较差）

### 2. 模块通过率
按模块统计通过率，降序排列，低通过率模块标红。

### 3. 优先级分析
按 P0/P1/P2 统计通过率，关注 P0 通过率。

### 4. 测试维度分析
按测试维度（正向/负向/边界/异常/性能/安全）统计通过率。

### 5. 失败用例分析
列出所有失败用例，包含：
- 用例编号、标题、所属模块
- 失败原因分类（功能缺陷/环境问题/数据问题/用例缺陷）
- 影响范围
- 修复建议

### 6. 阻塞用例分析
列出所有阻塞用例及阻塞原因。

### 7. 风险评估
基于失败和阻塞情况，评估整体风险等级：
- **高风险**：P0 用例失败、核心模块通过率 < 70%
- **中风险**：P1 用例失败、模块通过率 70%-90%
- **低风险**：P2 用例失败、模块通过率 > 90%

### 8. 测试结论与建议
- 是否可以发布
- 需要修复的问题
- 后续测试建议

## 质量评级标准

| 通过率 | 评级 | 说明 |
|--------|------|------|
| ≥ 95% | 优秀 🏆 | 质量优秀，可以发布 |
| 85%-95% | 良好 ✅ | 质量较好，修复少量问题后可发布 |
| 70%-85% | 中等 ⚠️ | 质量一般，需要修复较多问题 |
| < 70% | 较差 ❌ | 质量不达标，不建议发布 |

## 失败原因分类

| 类别 | 说明 | 示例 |
|------|------|------|
| 接口/服务异常 | 后端接口或服务问题 | 超时、502、503、空指针 |
| 权限/认证问题 | 鉴权、Token、越权 | 未授权访问、Token 过期 |
| 数据校验失败 | 前后端校验不一致 | 必填项遗漏、格式不符 |
| 状态流转错误 | 状态机配置问题 | 状态跳转不正确 |
| 并发/竞态 | 并发控制缺陷 | 超卖、重复扣减 |
| 边界值问题 | 边界条件处理不当 | off-by-one、溢出 |
| 兼容性 | 编码/浏览器/版本不兼容 | GBK 乱码、API 版本差异 |
| 环境配置 | 环境变量/配置项问题 | SSL 证书、端口配置 |
| 功能缺陷 | 系统功能实现与预期不符 | 下单后金额计算错误 |
| 数据问题 | 测试数据问题导致 | 数据准备不充分、数据冲突 |
| 用例缺陷 | 测试用例本身的问题 | 预期结果错误、步骤有误 |

## Python 脚本说明

### ⚠️ 运行环境（重要）

脚本依赖 `openpyxl`，该库**仅安装在 Hermes venv 中**。必须用 Hermes venv Python 运行：

```bash
HERMES_PYTHON="$HOME/.hermes/hermes-agent/venv/bin/python"
$HERMES_PYTHON scripts/generate_report.py testcases.xlsx -o test_report.md
```

### 脚本功能
- 脚本位置：`scripts/generate_report.py`
- 依赖：`openpyxl`（已在 Hermes venv 中安装）
- 功能：
  - 读取 Excel 测试用例文件，解析每行数据
  - 识别「执行结果」列（支持多种列名）
  - 按模块、优先级、测试维度统计通过率
  - 提取失败和阻塞用例
  - 生成结构化 Markdown 报告
  - 自动风险评估和发布建议

## 注意事项

1. 测试用例文件必须包含执行结果信息（通过/失败/阻塞等）
2. 如果执行结果列为空，会提示用户先执行测试
3. 报告中的风险评估仅供参考，实际风险评估需要结合业务场景
4. 发布建议仅供参考，最终决策需要人工判断
5. Markdown 表格支持在飞书/钉钉/企微等平台直接渲染
