# 生成测试用例 Skill

基于测试点清单，生成结构化的测试用例文件，支持 Excel 和 XMind 两种格式输出。

## 触发条件

当用户说以下内容时，使用本 Skill：
- "生成测试用例"
- "生成 Excel 用例"
- "生成 XMind 用例"
- "把测试点转成用例"
- 带有 "generate-testcases"、"测试用例生成" 等关键词

## 执行步骤

### 1. 读取测试点文件
使用 `read_file` 工具读取 `testpoints.md` 文件。

### 2. 确认输出格式

支持两种格式：

**Excel 格式（默认）：**
```
用户：生成测试用例，Excel 格式
用户：生成 Excel 用例
```

**XMind 格式：**
```
用户：生成测试用例，XMind 格式
用户：生成 XMind 用例
```

### 3. 确认测试维度过滤（可选）

可以按测试维度过滤：
```
# 只生成正向测试用例
用户：生成用例，只包含正向测试

# 只生成基础测试用例（正向、负向、边界、异常）
用户：生成用例，只包含基础测试

# 包含所有测试维度
用户：生成用例，包含性能和安全测试
```

### 4. 生成测试用例

#### 4.1 Excel 格式

使用 Python 脚本生成 `.xlsx` 文件。

**Excel 列结构：**

| 列 | 字段名 | 说明 |
|----|--------|------|
| A | 用例编号 | TC-001 格式，自动编号 |
| B | 所属模块 | 功能模块名称 |
| C | 功能点 | 功能点名称 |
| D | 测试维度 | 正向/负向/边界/异常/性能/安全 |
| E | 用例标题 | 简洁的用例描述 |
| F | 优先级 | P0/P1/P2 |
| G | 前置条件 | 执行用例前的必要条件 |
| H | 测试步骤 | 编号步骤，每步一行 |
| I | 测试数据 | 具体的测试数据 |
| J | 预期结果 | 具体的预期结果 |
| K | 备注 | 额外说明（可选） |

**Excel 格式化要求：**
- 表头加粗，背景色为深蓝色，白色字体
- 列宽自适应
- 优先级为 P0 的行用红色标注
- 优先级为 P1 的行用橙色标注
- 冻结首行

#### 4.2 XMind 格式

使用 Python 脚本生成 `.xmind` 文件。

**XMind 结构：**

```
[项目名称] 测试用例
├── [模块1名称]
│   ├── [功能点1.1]
│   │   ├── 正向测试
│   │   │   ├── TC-001: [用例标题] [P0]
│   │   │   │   ├── 前置条件: ...
│   │   │   │   ├── 测试步骤: ...
│   │   │   │   ├── 测试数据: ...
│   │   │   │   └── 预期结果: ...
│   │   │   └── TC-002: [用例标题] [P1]
│   │   ├── 负向测试
│   │   │   └── TC-003: ...
│   │   └── 边界测试
│   │       └── TC-004: ...
│   └── [功能点1.2]
│       └── ...
├── [模块2名称]
│   └── ...
└── 统计信息
    ├── 总用例数: N
    ├── P0: N
    ├── P1: N
    └── P2: N
```

### 5. 用例生成规则

#### 5.1 用例编号规则
- 格式：`TC-XXX`（三位数字，从001开始）
- 按模块和功能点顺序编号
- 编号连续不跳号

#### 5.2 优先级分配规则

**P0（高优先级）：**
- 核心业务流程的正向测试
- 涉及资金/安全的测试
- 关键功能的异常测试

**P1（中优先级）：**
- 一般功能的正向测试
- 大部分负向测试
- 关键功能的边界测试

**P2（低优先级）：**
- 非核心功能的测试
- 性能测试（除非特别要求）
- UI 细节测试

#### 5.3 用例标题规则
- 格式：`[操作对象] + [操作动作] + [预期/场景]`
- 示例：
  - ✅ `用户使用正确账号密码登录成功`
  - ✅ `库存不足时下单提示库存不足`
  - ❌ `验证下单功能`
  - ❌ `下单测试`

#### 5.4 测试步骤规则
- 编号格式：`1. 2. 3.`（每个步骤一行）
- 每个步骤必须包含具体操作
- 每个步骤必须包含操作对象（按钮/链接/输入框）
- 示例：
  - ✅ `1. 在用户名输入框输入"testuser"`
  - ✅ `2. 在密码输入框输入"123456"`
  - ✅ `3. 点击"登录"按钮`
  - ❌ `1. 登录系统`

#### 5.5 预期结果规则
- 必须包含明确的判断标准
- 必须包含具体的提示信息（如有）
- 必须包含具体的状态变更（如有）
- 示例：
  - ✅ `登录成功，页面跳转到首页，右上角显示用户名"testuser"`
  - ✅ `提示"密码错误，请重新输入"，停留在登录页面`
  - ❌ `登录成功`
  - ❌ `提示错误`

## 输入参数

在执行时需要确认：
- 测试点文件路径（必需，默认为 `testpoints.md`）
- 输出格式（可选，默认为 Excel）
- 输出目录（可选，默认为当前目录）
- 测试维度过滤（可选，默认为全部）
- 输出文件名（可选，默认为 `testcases.xlsx` 或 `testcases.xmind`）

## Python 脚本说明

### ⚠️ 运行环境（重要）

脚本依赖 `openpyxl`，该库**仅安装在 Hermes venv 中**，系统 `python3` 没有它。必须用 Hermes venv Python 运行：

```bash
HERMES_PYTHON="$HOME/.hermes/hermes-agent/venv/bin/python"
$HERMES_PYTHON scripts/generate_excel.py testpoints.md -o testcases.xlsx
$HERMES_PYTHON scripts/generate_xmind.py testpoints.md -o testcases.xmind
```

如果系统 `python3` 报 `ModuleNotFoundError: No module named 'openpyxl'`，不是安装问题——切换到 venv Python 即可。

### Excel 生成脚本
- 脚本位置：`scripts/generate_excel.py`
- 依赖：`openpyxl`（已在 Hermes venv 中安装）
- 功能：
  - 读取测试点 Markdown 文件
  - 解析模块、功能点、测试维度、测试点
  - 生成结构化的 Excel 文件（12列，含执行结果列）
  - 自动格式化：表头深蓝白字、P0红/P1橙/P2绿行标、冻结首行、自动筛选
  - 支持维度过滤：`-d positive,negative,basic,all`（中英文均可）

### XMind 生成脚本
- 脚本位置：`scripts/generate_xmind.py`
- 依赖：**无外部依赖**（仅用 Python 标准库 `zipfile` + `json`）
- 技术原理：`.xmind` 文件本质是 zip 包，内含 `content.json`。脚本直接构建 JSON 树结构并打包，不需要安装 `xmind` pip 包。
- 功能：
  - 读取测试点 Markdown 文件
  - 生成树状脑图：项目 → 模块 → 功能点 → 测试维度 → 用例
  - 每个用例节点包含前置条件、测试数据、预期结果子节点
  - 末尾自动生成统计节点（P0/P1/P2 分布）

## 注意事项

1. 测试用例基于测试点生成，测试点的质量直接影响用例质量
2. 生成的用例建议使用 test-case-review Skill 进行评审
3. Excel 文件需要使用 Excel 或 WPS 打开
4. XMind 文件需要使用 XMind 软件打开
5. 生成后可根据实际情况手动调整用例内容
6. 测试数据为示例数据，实际执行时需要根据环境调整
